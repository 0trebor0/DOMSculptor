## Non-Negotiable Rules

- Read relevant files before editing.
- Make the smallest safe change.
- Only edit files related to the task.
- Match existing style, naming, and patterns.
- Use existing helpers and utilities.
- Do not refactor unrelated code.
- Do not change APIs, routes, config, or behavior unless required.
- Do not add dependencies unless needed.
- Do not edit secrets, `.env`, tokens, keys, or credentials.
- Keep security checks.
- Add/update tests when behavior changes.
- Run relevant tests when possible; say why if not.
- Do not commit unless asked.
- Do not change git history.
- Do not modify lockfiles unless dependencies changed.
- Do not invent files, APIs, or requirements. Inspect first.

Final response: summarize changes, files changed, tests run, and anything not done.
